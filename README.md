# Kaoyan 2027 Coach（考研 2027 全科答疑教练）

考研 2027 全流程 AI 学习教练，覆盖建档、答疑、错题归档与复习、今日/周计划、学习日志、周月复盘和模考校准。

核心特点：

- 面向数学一、408、政治、英语一四科
- 深度集成 Obsidian，所有学习状态外置到本地 Markdown
- 通过脚本保证建档、归档、复习和复盘流程稳定可复用
- 兼容 `SKILL.md` 体系，适合接入 Codex、Claude Code、Gemini CLI、Antigravity 等工具

## 这份 README 适合谁

如果你想解决下面任一问题，这份文档就是给你准备的：

- 第一次安装这个 skill，不知道从哪里开始
- 想知道日常应该用哪些指令，顺序怎么走
- 想确认每次使用后，文件会写到 Obsidian 的哪里
- 想维护这个 skill，但不想先把整个代码库读一遍

正常使用时，你不需要手动执行 `scripts/` 里的 Python 脚本。  
你只需要在支持 skills 的 AI 工具里用 `/load`、`/wrong`、`/plan_today` 这类对话指令即可；脚本由 agent 在后台调用。

## 前置条件

- 本机可用的 `python3`
- 一个准备存放考研资料的 Obsidian vault 路径
- 一个支持 `SKILL.md`/skills 机制的 AI 工具

## 3 分钟上手

### 1. 安装 skill

在当前目录执行：

```bash
bash setup.sh
```

这个脚本会：

- 检查 `python3`
- 为 `~/.claude/skills/` 和 `~/.codex/skills/` 创建软链接
- 提示你配置 `KAOYAN_OBSIDIAN_ROOT`

如果你想手动安装，也可以：

```bash
ln -sf ~/.gemini/antigravity/skills/kaoyan-2027-coach ~/.claude/skills/kaoyan-2027-coach
ln -sf ~/.gemini/antigravity/skills/kaoyan-2027-coach ~/.codex/skills/kaoyan-2027-coach
```

### 2. 配置 Obsidian vault 路径

把下面这行加入 `~/.zshrc` 或 `~/.bashrc`：

```bash
export KAOYAN_OBSIDIAN_ROOT="/path/to/your/obsidian/vault/Kaoyan_2027_Prep"
```

然后重新加载 shell：

```bash
source ~/.zshrc
```

说明：

- `KAOYAN_OBSIDIAN_ROOT` 指向你的考研 Obsidian 仓库根目录
- 如果目录不存在，首次 `/load` 时会自动初始化所需结构

### 3. 在聊天里开始使用

首次进入支持 skill 的 AI 工具后，直接发送：

```text
/load
```

如果是第一次使用、档案还不存在，可以直接把基础信息一起发出去：

```text
/load 我准备考某大学计算机，目标总分 360，考试日期 2026-12-20，每天可投入 6 小时，现在处于 408 重构期。
```

agent 会自动：

- 初始化 Obsidian 目录
- 创建 `我的学习者档案.md`
- 生成四科知识地图模板
- 给出当前阶段、风险提醒和立刻开始建议

## 推荐使用流程

### 首次建档

1. 用 `/load` 建立基础档案
2. 确认目标院校/专业、目标分数、考试日期和每日时长
3. 开始正常提问或录入错题

### 日常学习闭环

推荐的顺序是：

1. `/load` 恢复上下文和优先事项
2. `/plan_today 6h` 生成今日计划
3. 遇到错题就用 `/wrong`
4. 学完当天内容后用 `/progress`
5. 有到期卡片时用 `/review`

### 每周和每月节奏

- 每周开始时用 `/plan_week 35h`
- 每周结束时用 `/recap week`
- 每月结束时用 `/recap month`
- 每次模考后用 `/recalibrate 政治=62 数学一=118 英语一=80 408=95`

## 常用指令

| 指令 | 作用 | 典型场景 |
|------|------|----------|
| `/load` | 恢复学习上下文；首次使用时自动建档 | 开始今天学习前先看当前阶段和风险 |
| `/reset [hard]` | 重置测试数据 | 想把试跑数据清掉，正式开始使用 |
| `/wrong [科目] [题目]` | 解析并归档错题 | 做错一道数学/408/政治/英语题后立即记录 |
| `/review` | 扫描到期错题并复习 | 清理积压旧题、做 SRS 复习 |
| `/plan_today [时长]` | 生成今日学习计划 | 今天有 5 小时，想知道怎么分配 |
| `/plan_week [总时长]` | 生成本周计划 | 周初排整周节奏 |
| `/progress [内容]` | 写学习日志并沉淀训练结果 | 晚上收尾，记录今天学了什么、卡在哪里 |
| `/recap [week\|month]` | 做周/月复盘 | 想看趋势、产出、卡点和下一步 |
| `/test [章节]` | 按知识地图抽测 | 想测“操作系统进程同步”是否真的会了 |
| `/recalibrate ...` | 记录模考并校准策略 | 模考之后快速更新学习策略 |
| `/mock [科目] [题量]` | 对话式限时训练 | 临时做一轮小测或变式训练 |

补充说明：

- `/wrong`、`/review`、`/test` 会推动知识地图更新
- `/progress` 主要负责学习日志、结构化成绩和稳定短板回写
- 解释概念、知识串联、解题纠错、Anki 卡片生成等，也可以直接自然语言对话，不一定非要用斜杠指令

## 直接可用的输入示例

### 错题归档

```text
/wrong 数学一 这道中值定理题我一开始想直接证单调，但不知道怎么把题里的 f''(x)>0 用起来，最后卡住了。
```

```text
/wrong 408 操作系统这题我把用户态和内核态切换条件搞混了，选了 C。
```

### 今日计划

```text
/plan_today 6h
```

### 学习收尾

```text
/progress 今天做完了线代特征值专题和 408 操作系统调度，数学选择题 12/15，错在特征向量判断不稳。
```

### 知识测试

```text
/test 进程同步
```

### 模考校准

```text
/recalibrate 政治=62 数学一=118 英语一=80 408=95
```

## 文件会写到哪里

首次初始化后，默认会形成下面这套结构：

```text
Kaoyan_2027_Prep/
├── 我的学习者档案.md
├── 知识地图/
│   ├── 数学一.md
│   ├── 408.md
│   ├── 政治.md
│   └── 英语一.md
├── 学习日志/
├── 周计划/
├── 错题本/
├── 复盘报告/
└── 知识笔记/
```

主要输出位置如下：

- `/load`：读取 `我的学习者档案.md`、最新学习日志、最新复盘报告
- `/wrong`：写入 `错题本/[科目]/[章节]/`
- `/review`：更新已有错题卡，并回写知识地图
- `/plan_today`：生成今日计划内容
- `/plan_week`：写入 `周计划/YYYY-Www.md`
- `/progress`：写入 `学习日志/YYYY-MM-DD.md`
- `/recap week|month`：写入 `复盘报告/`
- `/recalibrate`：写入 `我的学习者档案.md` 和 `复盘报告/YYYY-MM-DD-模考分析.md`

## 适合怎样的使用习惯

这个 skill 最适合下面这种节奏：

- 平时用自然语言提问，让 AI 把题讲明白
- 遇到真正值得沉淀的错误时，用 `/wrong` 落盘
- 每天结束时用 `/progress` 统一归档
- 每周做一次 `/recap week`，每月或模考后做一次策略校准

这样可以让档案、知识地图、错题本、日志和复盘报告形成闭环，不会出现“聊完就丢”的问题。

## 常见问题

### 我需要自己运行 Python 脚本吗？

不需要。对普通使用者来说，脚本是后台实现细节。你只要在对话里发指令，agent 会自动调用脚本。

### 首次使用没有 Obsidian 目录怎么办？

直接 `/load` 即可。只要 `KAOYAN_OBSIDIAN_ROOT` 配好了，缺失目录和模板会自动创建。

### 我可以只用自然语言，不用这些斜杠指令吗？

可以，但推荐把“需要落盘的动作”交给结构化指令：

- 讲题、解释概念、做知识串联：自然语言即可
- 建档、错题归档、复习、计划、日志、复盘：优先使用 `/load`、`/wrong`、`/review`、`/plan_today`、`/progress` 等指令

### 这个 skill 支持哪些科目？

当前内置支持：

- 数学一
- 408
- 政治
- 英语一

### 如果我换了 vault 目录怎么办？

更新 `KAOYAN_OBSIDIAN_ROOT`，然后重新打开终端或执行 `source ~/.zshrc`。

## 仓库结构速览

如果你是维护者，可以先看这几个目录：

- `SKILL.md`：skill 的核心行为说明和工作流规则
- `scripts/`：确定性脚本，负责建档、归档、复盘、测试等落盘操作
- `templates/`：Obsidian 模板
- `references/`：数学一和 408 的专项答疑参考
- `tests/`：主要脚本的回归测试

## 维护者工作流

如果你后面还会继续迭代这个 skill，推荐按下面的顺序改：

1. 先改 `SKILL.md`
2. 再改 `scripts/` 或 `templates/`
3. 最后补 `README.md` 和 `tests/`

这样做的原因是：

- `SKILL.md` 决定 agent 的行为边界和调用方式
- `scripts/` 决定可重复执行的确定性能力
- `templates/` 决定最终落盘的 Markdown 结构
- `README.md` 负责告诉人怎么用，不应该先于真实能力“超前承诺”

几类常见改动，建议这样处理：

- 新增指令：同时更新 `SKILL.md`、对应脚本、README 的指令表和示例
- 调整落盘格式：同时检查 `templates/`、脚本输出和已有测试
- 修改答疑策略：先改 `SKILL.md`，必要时把长说明拆到 `references/`
- 新增学科支持：至少补齐知识地图模板、脚本分支、README 说明和回归测试

提交前建议快速自查这 5 件事：

1. README 里的指令名、参数名是否和 `SKILL.md` 一致
2. 有没有把“用户对话入口”和“维护脚本入口”混写在一起
3. 模板字段名改了以后，解析脚本和测试是否同步
4. 是否还保留了错误的旧路径、旧文件名或旧命令
5. 如果能力变了，示例输入和输出位置说明有没有一起更新

## 开发与验证

运行测试：

```bash
python3 -m pytest tests/ -v
```

如果你修改了：

- `SKILL.md`：建议同步检查 README 的用户用法是否还准确
- `templates/`：建议至少跑相关测试，并手动看一次生成 Markdown 的可读性
- `scripts/`：建议补或改对应 `tests/`

## 设计原则

- 档案是唯一事实源，个人信息不硬编码在 skill 里
- 错题驱动知识地图更新，避免只聊天不沉淀
- 轻量归档，单次错题和每日总结分开收口
- 复盘和模考校准可追踪，方便长期策略调整
- 对考频和真题来源保持保守表达，避免伪精确和幻觉
