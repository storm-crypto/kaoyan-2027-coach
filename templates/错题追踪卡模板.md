# 错题追踪卡模板

## Obsidian 错题追踪卡格式

文件命名规则：`错题本/[科目]/[章节]/[考点关键词]-[来源简称].md`
例：`错题本/数学一/多元积分/二重积分极坐标-900题.md`

**文件名即唯一标识**，旧题识别通过搜索文件名实现，无需额外 ID 字段。

```markdown
---
source:
topic:
error_tags: []
first_wrong_at: YYYY-MM-DD
last_review_at: YYYY-MM-DD
wrong_count: 1
status: 不会
next_review: YYYY-MM-DD
review_interval: 1
---

#subject/math1 #topic/多元积分 #status/不会 #source/900题

## [考点关键词] — [来源简称]

### 错误原因
- [这次错在什么地方]

### 正确思路 / 核心结论
- [本题正确思路、突破口、关键结论]

### 易错点 / 变式提醒
- [下次要防的坑]

### 历史记录
- YYYY-MM-DD - 不会 / 半会 / 会 - [一句话简评]
```

## 间隔复习规则

每张错题卡有 `next_review`（下次复习日期）和 `review_interval`（当前间隔天数）。

**新建时：** `next_review` = 明天，`review_interval` = 1

**复习后根据结果调整：**
| 结果 | interval 变化 | 说明 |
|------|--------------|------|
| 不会 | 重置为 1 天 | 明天再来 |
| 半会 | 保持不变 | 同样间隔再练一次 |
| 会 | × 2（上限 30 天） | 1→2→4→8→16→30 |

`next_review` = 今天 + 新 interval

**到期未复习的题自动降级：** 超过 `next_review` 7 天以上未复习的卡，`review_interval` 重置为 1。

## 模板使用规则

- 第一次出现该题时，新建完整错题追踪卡，`wrong_count` 从 `1` 开始，`next_review` 设为明天，`review_interval` 设为 1。
- 复习同一道题时，不新建平行条目；更新 `last_review_at`、`wrong_count`、`status`、`next_review`、`review_interval`，在"历史记录"中追加一条新记录。
- 日期格式：`YYYY-MM-DD`。
- 同类错因但不是同一道题，用 `error_tags` 关联，不强制合并。
- `status` 固定为 `不会 / 半会 / 会` 三档。

## Tags 用法

- **正文 tag**：只保留 `#subject/...`、`#topic/...`、`#status/...`、`#source/...`，不超过 5 个
- **YAML `error_tags`**：保存完整错因描述，用于跨题搜索同类错误

## 推荐输入模板

### 首次建卡

```text
/wrong_math
来源：900题
题目：[题目内容或截图]
我的错误思路：[怎么想的、选了什么]
```

### 复习旧错题

```text
/wrong_math
旧题复习
来源：900题
这次结果：不会 / 半会 / 会
题目：[题目内容或截图]
我这次的思路：[现在怎么想的]
```
