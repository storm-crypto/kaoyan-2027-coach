## 这个 skill 是做什么的

Kaoyan 2027 Coach 是一个面向 2027 考研用户的 AI 学习教练 skill，目标不是“临时帮你答一道题”，而是把你的学习状态长期沉淀到本地 Obsidian 仓库中，让以下动作形成闭环：

- 用 `/load` 恢复当前阶段、风险提醒和下一步动作
- 用 `/wrong` 把真正值得沉淀的错题解析并归档
- 用 `/review` 做间隔复习，而不是只建卡不回顾
- 用 `/plan_today` 和 `/plan_week` 管理学习节奏
- 用 `/progress` 写学习日志并沉淀训练结果
- 用 `/recap` 做周/月复盘
- 用 `/recalibrate` 记录模考并调整策略

当前支持的学科包括：

- 数学一
- 408
- 政治
- 英语一

除了结构化指令，你也可以直接用自然语言提问，例如：

- “帮我解释一下拉格朗日中值定理和柯西中值定理的区别”
- “这道 408 选择题我为什么会被 C 选项误导”
- “给我把这段英语阅读长难句拆开”
- “把这个知识点转成 Anki 卡片”

但是如果你希望内容真正写回本地仓库，推荐优先使用结构化指令。

## 使用前需要准备什么

开始前建议确认下面三件事：

- 你的环境里有 `python3`
- 你有一个用于存放考研资料的 Obsidian vault 路径
- 你使用的 AI 工具支持 `SKILL.md`/skills 机制

### 安装 skill

推荐直接执行仓库根目录下的安装脚本：

```bash
bash setup.sh
```

这个脚本会：

- 检查本机是否安装了 `python3`
- 为 `~/.claude/skills/kaoyan-2027-coach` 创建软链接
- 为 `~/.codex/skills/kaoyan-2027-coach` 创建软链接
- 提示你配置 `KAOYAN_OBSIDIAN_ROOT`

如果你不想跑脚本，也可以手动安装：

```bash
ln -sf ~/.gemini/antigravity/skills/kaoyan-2027-coach ~/.claude/skills/kaoyan-2027-coach
ln -sf ~/.gemini/antigravity/skills/kaoyan-2027-coach ~/.codex/skills/kaoyan-2027-coach
```

### 配置环境变量

把下面一行加入你的 shell 配置文件：

```bash
export KAOYAN_OBSIDIAN_ROOT="/path/to/your/obsidian/vault/Kaoyan_2027_Prep"
```

常见位置：

- `~/.zshrc`
- `~/.bashrc`

保存后重新加载：

```bash
source ~/.zshrc
```

或者重开一个终端窗口。

这个环境变量表示：

- 你的 Obsidian 考研仓库根目录在哪里
- 这个 skill 生成的档案、日志、错题卡、复盘报告应该写到哪里

### 普通使用者需要手动跑脚本吗

不需要。

对普通用户来说：

- `scripts/` 目录里的脚本属于内部实现
- 正常使用入口是和 AI 对话
- 只要发送 `/load`、`/wrong`、`/progress` 这类指令即可

只有在你自己维护这个仓库、调试脚本、写测试或者排查问题时，才需要直接运行 `scripts/*.py`。

## 核心工作方式

这个 skill 的核心理念是“把学习状态外置到文件，而不是停留在聊天记录里”。

你可以把它理解成 5 个层次：

- `我的学习者档案.md`
  这是长期摘要，记录你的目标、阶段、短板、下一步建议等关键信息
- `知识地图/*.md`
  这是各学科的掌握状态视图，重点反映“不会 / 半会 / 会”
- `错题本/`
  这是具体题卡的沉淀区，每张卡记录题目、选项、错因、解法和复习状态
- `学习日志/`
  这是每天收尾时写入的学习记录
- `周计划/` 和 `复盘报告/`
  这是节奏安排和阶段性复盘的输出位置

### 它不是怎样工作的

下面这些理解容易出偏：

- 它不是单纯的聊天助手。它的价值在于“讲解 + 落盘 + 跟踪”
- 它不是只管错题。它同时也管计划、日志、复盘和模考校准
- 它不是强制你每一步都用命令。概念解释、知识串联、Anki 卡片等依然可以自然语言完成
- 它不是让你手写 Markdown 模板。正常情况下，目录和模板都由脚本自动创建和更新

## 首次使用流程

第一次使用时，推荐按下面顺序来：

1. 配好 `KAOYAN_OBSIDIAN_ROOT`
2. 在 AI 工具里发送 `/load`
3. 如果还没有基础档案，就把目标院校/专业、目标总分、考试日期、每日时长、当前阶段一起发给它
4. 等它初始化 Obsidian 目录和模板
5. 再开始正常提问、计划和错题归档

### 首次使用示例

最简单的写法：

```text
/load
```

更推荐的写法：

```text
/load 我准备考某大学计算机，目标总分 360，考试日期 2026-12-20，每天可投入 6 小时，现在处于 408 重构期。
```

执行后通常会发生这些事：

- 创建缺失的目录
- 生成 `我的学习者档案.md`
- 生成四科知识地图模板
- 读取档案和最近记录，输出当前阶段、优先问题、风险提醒和立刻开始建议

### 首次初始化后会看到哪些目录

```text
Kaoyan_2027_Prep/
├── 我的学习者档案.md
├── 知识地图/
│   ├── 数学一.md
│   ├── 408.md
│   ├── 政治.md
│   └── 英语一.md
├── 学习日志/
├── 周计划/
├── 错题本/
├── 复盘报告/
└── 知识笔记/
```

如果这些目录之前不存在，首次 `/load` 会自动建立。

## 推荐学习工作流

这个 skill 最适合按节奏使用，而不是“偶尔想起来才记一笔”。下面是一套推荐的日常工作流。

### 日常流程

1. 开始学习前先 `/load`
2. 然后 `/plan_today 6h` 生成今天的主线安排
3. 学习过程中，真正值得沉淀的错题用 `/wrong`
4. 当天结束时用 `/progress` 统一写日志
5. 如果有到期卡片，用 `/review` 清理旧题

对应示例：

```text
/load
/plan_today 6h
/wrong 数学一 这道题我卡在第一步
/progress 今天做了线代特征值专题和操作系统调度
/review
```

### 每周流程

每周建议至少完成两件事：

- 周初用 `/plan_week 35h`
- 周末用 `/recap week`

这样周计划和周复盘可以形成闭环，不会只排计划不看结果。

### 每月或模考后流程

每月或每次模考后，推荐使用：

- `/recap month`
- `/recalibrate 政治=62 数学一=118 英语一=80 408=95`

这两个动作适合用来重新校准资源分配和阶段目标。

## 指令详解

这一节会按“什么时候用、该怎么说、会写到哪里、要注意什么”来说明每个指令。

### `/load`

作用：

- 恢复你的当前学习上下文
- 首次使用时自动初始化档案和目录结构

什么时候用：

- 每天开始学习前
- 刚换了 AI 工具或新开会话时
- 想快速恢复最近学习状态时

推荐输入：

```text
/load
```

如果还没有档案，推荐直接带上基础信息：

```text
/load 我准备考某大学计算机，目标总分 360，考试日期 2026-12-20，每天可投入 6 小时，现在处于 408 重构期。
```

会做什么：

- 读取 `我的学习者档案.md`
- 读取最新学习日志
- 读取最新复盘或模考报告
- 输出当前阶段、风险提醒和立刻开始建议

会写到哪里：

- 首次使用时会创建 `我的学习者档案.md` 和知识地图模板

注意：

- `/load` 不是简单的“打招呼”，它是恢复上下文的主入口
- 如果环境变量没有配好，首次建档会失败

### `/reset [hard]`

作用：

- 清理测试数据，让你重新开始使用这套系统

什么时候用：

- 你试跑过一轮，想从更干净的状态重新开始
- 你确认之前写进去的是无效演示数据

示例：

```text
/reset
```

```text
/reset hard
```

区别：

- `/reset` 默认保留基础建档信息
- `/reset hard` 会把基础信息也恢复为空白模板

注意：

- 这是破坏性操作
- 默认不会清空 `知识笔记/`
- 如果你已经正式使用了一段时间，执行前建议先确认

### `/wrong [科目] [题目及错解]`

作用：

- 先把题讲清楚
- 再把题归档进 `错题本/`
- 同时更新知识地图掌握度

什么时候用：

- 做错一道真正值得回头复习的题
- 做对了但方法不稳、解释不清，也值得归档

推荐输入方式：

```text
/wrong 数学一 这道中值定理题我一开始想直接证单调，但不知道怎么把题里的 f''(x)>0 用起来，最后卡住了。
```

```text
/wrong 408 操作系统这题我把用户态和内核态切换条件搞混了，选了 C。
```

```text
/wrong 英语一 这道阅读我把作者态度题做错了，我把让步句当成主结论了。
```

最好提供的信息：

- 学科
- 题目内容或关键信息
- 你错在哪里
- 你卡在哪一步
- 如果是选择题，尽量给出选项

会做什么：

- 识别题型和考点
- 给出专项解析
- 指出错因和下次怎么避免
- 生成或更新错题卡
- 更新知识地图中的掌握状态

会写到哪里：

- `错题本/[科目]/[章节]/`
- `知识地图/[科目].md`

注意：

- 对数学一和 408，解释会更强调“第一步怎么想到的”和“为什么别的路不优先”
- 如果你发的是截图，仍然建议尽量补充题干和选项文字，后续复习体验会更好
- `/wrong` 主要负责“单题沉淀”，不会顺手替你写当天学习日志

### `/review`

作用：

- 扫描到期卡片
- 带你逐题回顾
- 根据复习结果更新 SRS 状态和掌握度

什么时候用：

- 想清掉积压的旧题时
- 不想让错题本只增不复习时

示例：

```text
/review
```

会做什么：

- 扫描所有到期错题
- 按科目分组展示
- 优先展示卡片中的题目和选项
- 根据你的回答更新卡片状态

会写到哪里：

- 更新已有错题卡
- 回写知识地图

注意：

- 如果到期旧题已经很多，应该优先清旧题，再继续堆新题
- `/review` 不会新建错题卡，它只处理已有卡片
- 在 Codex、Antigravity 这类对话框里，公式会优先以更易读的文本形式展示；错题卡文件中仍保留原始 LaTeX，保证 Obsidian 渲染不受影响

### `/plan_today [时长]`

作用：

- 生成当天学习计划

什么时候用：

- 想快速决定今天学什么、先后顺序怎么排
- 希望把到期复习任务和新内容安排整合在一起

示例：

```text
/plan_today 6h
```

会做什么：

- 读取档案里的当前重点
- 扫描到期错题
- 生成今日计划
- 提醒你当天结束后用 `/progress`

注意：

- 计划里会优先插入到期复习任务
- 如果可用时间比较短，建议直接给出一个明确数字，例如 `3h`、`4.5h`

### `/plan_week [总时长]`

作用：

- 生成本周学习计划

什么时候用：

- 周初做节奏规划时
- 想根据总时长而不是每天零散安排来做统筹时

示例：

```text
/plan_week 35h
```

会做什么：

- 读取档案中的重点问题
- 结合本周到期复习任务做科目分配
- 生成每日节奏和检查点

会写到哪里：

- `周计划/YYYY-Www.md`

### `/progress [今天学了什么]`

作用：

- 写当天学习日志
- 记录训练成绩
- 必要时回写稳定短板

什么时候用：

- 每天收尾时
- 一次专项训练或一轮真题结束后

示例：

```text
/progress 今天做完了线代特征值专题和 408 操作系统调度，数学选择题 12/15，错在特征向量判断不稳。
```

最好包含的信息：

- 今天学了什么
- 学了多久
- 哪些地方掌握得不错
- 哪些地方还卡住
- 如果做了训练，分数或正确率是多少

会写到哪里：

- `学习日志/YYYY-MM-DD.md`
- 必要时回写 `我的学习者档案.md`

注意：

- `/progress` 是“每日收尾”的主入口
- 如果你只在 `/wrong` 里沉淀错题、不用 `/progress`，长期来看日志和趋势会缺失

### `/recap [week|month]`

作用：

- 生成周复盘或月复盘

什么时候用：

- 每周结束时看产出、趋势和卡点
- 每月结束时回看整体阶段表现

示例：

```text
/recap week
```

```text
/recap month
```

会做什么：

- 汇总对应周期内的学习日志
- 汇总结构化成绩
- 汇总错题复习情况
- 输出卡点和下一步建议

会写到哪里：

- `复盘报告/YYYY-Www-周复盘.md`
- `复盘报告/YYYY-MM-月复盘.md`

### `/test [章节]`

作用：

- 基于知识地图生成结构化题单
- 检验你是不是“真的会了”

什么时候用：

- 学完一个章节后想查漏补缺
- 觉得自己“差不多懂了”，想做一次快速口头抽测

示例：

```text
/test 进程同步
```

```text
/test 线性代数 二次型
```

会做什么：

- 优先抽“不会”和空白考点
- 再补“半会”考点
- 给出 3-5 道题及判定要点
- 根据你的回答更新知识地图

注意：

- `/test` 只回写知识地图，不创建错题卡
- 如果你在测试中暴露出一题非常典型的错误，再额外转 `/wrong` 会更合适

### `/recalibrate 政治=62 数学一=118 英语一=80 408=95`

作用：

- 记录一次模考成绩
- 根据当前结果调整策略

什么时候用：

- 模考之后
- 做完一次比较完整的多科训练之后

示例：

```text
/recalibrate 政治=62 数学一=118 英语一=80 408=95
```

会做什么：

- 记录本次成绩
- 对比目标或上次成绩
- 输出相对变化和调整建议

会写到哪里：

- `我的学习者档案.md`
- `复盘报告/YYYY-MM-DD-模考分析.md`

### `/mock [科目] [题量]`

作用：

- 做一轮对话式限时训练

什么时候用：

- 想临时做一轮小测
- 想基于错题和短板快速刷变式

示例：

```text
/mock 数学一 5
```

注意：

- 这是启发式对话流程，不是完全脚本化接口
- 更适合“练一轮”而不是“做严格可回放评测”
- 如果其中某道题非常值得沉淀，可以再转 `/wrong`

### 自然语言提问

除了这些结构化指令，你还可以直接说：

- “帮我把这道题的错因总结成一句下次提醒”
- “把这段内容做成 5 张 Anki 卡片”
- “比较二叉排序树和平衡二叉树的使用场景”
- “这道英语阅读的干扰项分别错在哪”

建议的判断标准是：

- 需要写回系统：优先用结构化指令
- 只需要解释或临时辅助：自然语言即可

## 怎样和它配合得更好

这个 skill 会在你提供足够上下文时明显更好用。下面这些做法很有帮助。

### 录入错题时尽量提供这些信息

- 学科
- 题目或题目摘要
- 你选了什么
- 正确答案是什么
- 你为什么会错
- 你卡在哪一步

不一定每次都要齐全，但信息越完整，归档质量越高。

### 写 `/progress` 时尽量具体一点

比起：

```text
/progress 今天学了很多
```

更推荐：

```text
/progress 今天做完了线代特征值专题和 408 调度算法，数学选择题 12/15，卡在特征向量判断和时间片轮转的边界条件。
```

因为后者更有利于：

- 写出有用的学习日志
- 形成可复盘的短板记录
- 后续周/月复盘抓到真实趋势

### 不要把所有内容都丢给 `/wrong`

下面这些情况更适合自然语言或别的命令：

- 只是想问概念：
  直接自然语言
- 想记录今天整体学习情况：
  `/progress`
- 想抽测一个章节：
  `/test`
- 想做阶段回顾：
  `/recap`

### 建议坚持的最小闭环

如果你不想一开始就用全套流程，至少建议保留这三个动作：

1. `/load`
2. `/wrong`
3. `/progress`

这是最小但已经有价值的一条闭环链路。

## 常见问题

### 我必须每天都用所有指令吗

不需要。

最推荐的最小组合是：

- 学习前 `/load`
- 遇到典型错题用 `/wrong`
- 学习结束用 `/progress`

如果你状态更稳定了，再逐步加入 `/review`、`/plan_week`、`/recap`。

### 我平时只想问题，不想记日志，可以吗

可以。

但如果你长期不使用 `/progress`，那么：

- 日志会缺失
- 训练成绩趋势不完整
- 周/月复盘的信息也会变少

### 我可以完全不用斜杠命令吗

可以做一部分事，但不推荐完全这样用。

比较好的分工是：

- 解释概念、知识串联、临时问答：自然语言
- 建档、归档、计划、日志、复盘：结构化指令

### 如果我换了 Obsidian 路径怎么办

更新 `KAOYAN_OBSIDIAN_ROOT` 后，重新打开终端或执行：

```bash
source ~/.zshrc
```

### 首次使用没有目录会怎样

只要环境变量配置正确，首次 `/load` 会自动创建所需目录和模板。

### 我需要自己维护 Markdown 模板吗

普通用户不需要。

模板和字段由仓库内脚本统一控制。只有你准备改这个 skill 本身时，才需要关心 `templates/` 目录。

### 我可以把历史笔记一起放进这个 vault 吗

可以。

这个 skill 默认不会主动删除你的 `知识笔记/`。即使执行 `/reset`，默认也不会清空这部分内容，除非你明确要求。

### `/wrong` 和 `/test` 的区别是什么

可以这样理解：

- `/wrong`：
  适合单题深入解析和归档
- `/test`：
  适合按章节抽测和更新掌握状态

如果一轮 `/test` 暴露出某道特别典型的错误，再把那道题转成 `/wrong`，会更有价值。
