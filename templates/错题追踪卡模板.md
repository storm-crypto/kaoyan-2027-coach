# 错题追踪卡模板

## Obsidian 错题追踪卡格式

```markdown
---
source:
question_id:
topic:
error_tags: []
first_wrong_at: YYYY-MM-DD HH:mm
last_review_at: YYYY-MM-DD HH:mm
wrong_count: 1
status: 不会
next_review: YYYY-MM-DD
review_interval: 1
---

#subject/math1 #topic/多元积分 #status/不会 #source/1000题 #error/第一步联想慢

## 题目标题 / 题号

### 题目来源
- [书名 / 套卷 / 年份 / 题号]

### 错误原因
- [这次错在什么地方]

### 正确思路 / 核心结论
- [本题正确思路、突破口、关键结论]

### 易错点 / 变式提醒
- [下次要防的坑]

### 本次复习简评
- [一句话说明这次为什么还不会、半会，或已经会做]

### 历史记录
- YYYY-MM-DD HH:mm - 不会 / 半会 / 会做 - [本次复习简评]
```

## 间隔复习规则

每张错题卡有 `next_review`（下次复习日期）和 `review_interval`（当前间隔天数）。

**新建时：** `next_review` = 明天，`review_interval` = 1

**复习后根据结果调整：**
| 结果 | interval 变化 | 说明 |
|------|--------------|------|
| 不会 | 重置为 1 天 | 明天再来 |
| 半会 | 保持不变 | 同样间隔再练一次 |
| 会做 | × 2（上限 30 天） | 1→2→4→8→16→30 |

`next_review` = 今天 + 新 interval

**到期未复习的题自动降级：** 超过 `next_review` 7 天以上未复习的卡，`review_interval` 重置为 1。

## 模板使用规则

- 第一次出现该题时，新建完整错题追踪卡，`wrong_count` 从 `1` 开始，`next_review` 设为明天，`review_interval` 设为 1。
- 复习同一道题时，不新建平行条目；更新 `last_review_at`、`wrong_count`、`status`、`next_review`、`review_interval` 和"本次复习简评"，在"历史记录"中追加一条新记录。
- 时间格式：`YYYY-MM-DD HH:mm`。
- 同类错因但不是同一道题，用 `error_tags` 关联，不强制合并。

## Tags 用法

- **正文 tag**：只保留高频检索维度 `#subject/...`、`#topic/...`、`#status/...`、`#source/...`，加 1-2 个最值得搜的 `#error/...`
- **YAML 字段**：精确记录完整错因，用于后续筛选
- `error_tags` 作为数组保存完整错因，正文 tag 不要堆太多，避免标签爆炸
- `status` 固定为 `不会 / 半会 / 会做` 三档，YAML 和正文 tag 各保留一份

## 推荐输入模板

### 首次建卡

**数学：**
```text
/wrong_math
来源：
题目：
我的错误思路 / 错误答案：
我卡住的地方：
```

**408：**
```text
/wrong_408
来源：
题目：
我的错误选项 / 错误思路：
我卡住的地方：
```

### 复习旧错题

**数学：**
```text
/wrong_math
这是之前那道旧题复习
来源：
这次结果：不会 / 半会 / 会做
这次简评：
题目：
我这次的做法 / 现在的思路：
```

**408：**
```text
/wrong_408

来源：

这次简评：

```
